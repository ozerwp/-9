# 实验9：单元测试

## 实验目标
1. 学习单元测试的基本概念和重要性
2. 掌握使用pytest框架进行单元测试的方法
3. 设计和实现全面的单元测试用例
4. 理解测试驱动开发(TDD)的基本思想

## 实验内容
1. 实现图书管理系统的核心模块
2. 使用pytest编写单元测试用例
3. 测试覆盖正常情况和异常情况
4. 确保代码质量和功能正确性

## 项目结构
```
experiment9/
├── book_management/
│   ├── models/
│   │   └── book.py          # 图书模型
│   ├── repositories/
│   │   └── book_repository.py  # 图书数据访问层
│   └── services/
│       └── book_service.py     # 图书业务逻辑层
├── test_book_service.py         # 单元测试脚本
└── README.md                    # 实验说明文档
```

## 核心功能
1. 图书添加
2. 图书查询
3. 图书更新
4. 图书删除
5. 图书借阅
6. 图书归还

## 测试覆盖
- 正常情况测试
- 异常情况测试
- 边界条件测试

## 运行测试
```bash
pytest test_book_service.py -v
```

## 实验要求
1. 遵循PEP8编码规范
2. 测试覆盖率达到80%以上
3. 使用断言验证测试结果
4. 编写清晰的测试用例描述
